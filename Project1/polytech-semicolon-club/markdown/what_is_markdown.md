# 마크다운(Markdown) 언어의 이해와 활용

## 1. 마크다운(Markdown)이란 무엇인가?
마크다운은 일반 텍스트 문서의 양식을 편집하는 문법입니다. 복잡한 HTML 태그를 사용하지 않고도 `#`, `*`, `-` 와 같은 직관적이고 간단한 특수기호를 사용하여 문서의 구조를 쉽게 정의할 수 있는 `가벼운 마크업 언어(Lightweight Markup Language)`입니다.

- **목적:** 읽고 쓰기 쉬운 텍스트 문서를 지향합니다.
- **특징:** 작성된 텍스트 자체로도 형태를 알아볼 수 있으며, 클릭 한 번으로 HTML 등의 웹 문서로 쉽게 변환됩니다.

---

## 2. 개발자들은 언제, 어떻게 사용하는가?
현대의 소프트웨어 개발 환경에서 마크다운은 원활한 협업과 기록을 위한 표준 도구로 자리 잡았습니다. 

1. **프로젝트 설명서 (README.md) 작성**
   - 깃허브(GitHub) 등 코드 저장소의 메인 화면에 프로젝트의 목적, 설치 방법, 실행 방법 등을 설명할 때 사용합니다.

2. **협업 도구에서의 의사소통**
   - 깃허브의 이슈(Issue) 및 PR(Pull Request) 작성, 슬랙(Slack)이나 디스코드(Discord) 등에서의 코드 및 로직 설명 시 활용합니다.

3. **기술 블로그 및 사내 위키(Wiki)**
   - 팀원들과 지식을 공유하는 시스템(예: Notion)이나 개인 기술 블로그(Velog, Tistory 등)에 학습 내용을 기록할 때 사용합니다. 

---

## 3. 왜 마크다운을 익혀야 하는가?
성공적인 프로젝트와 원활한 팀워크를 위해 마크다운 학습은 필수적입니다.

- **극강의 작성 효율성:** 마우스로 서식 버튼을 누를 필요 없이 키보드만으로 문서 구조를 짤 수 있어 개발 흐름이 끊기지 않습니다.

- **범용성과 호환성:** 거의 모든 개발 플랫폼과 협업 툴에서 마크다운을 기본으로 지원하므로, 한 번 익혀두면 어디서든 사용할 수 있습니다.

- **버전 관리(Git)의 용이성:** 일반 텍스트(.md) 파일이기 때문에, 워드나 한글 파일과 달리 Git을 통해 누가, 언제, 어느 부분을 수정했는지 코드처럼 정확하게 추적하고 관리할 수 있습니다.

> **💡 요약:** 좋은 코드를 짜는 것만큼, 그 코드를 남들이 알아보기 쉽게 **문서화**하는 것도 개발자의 핵심 역량입니다. 마크다운은 그 문서화를 위한 가장 강력하고 효율적인 무기입니다. 세미콜론 동아리 프로젝트, 수업시간 등의 활동에서부터 적극적으로 활용해 보시기 바랍니다. *특히 수업시간에 메모장이나 워드 등을 사용하지 않고 md 파일 (마크다운 언어로 작성된 파일)을 활용하면 화면을 바로 스크린샷 찍어서 사진을 넣으며 수업 내용을 필기하는 등에 용이하니 적극 사용 바랍니다.*
