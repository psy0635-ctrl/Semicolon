# 시작하기 전에: 핵심 개념 이해하기
초보자도 쉽게 이해할 수 있도록 비유를 통해 기본 개념을 먼저 정리합니다.

### Git
> 파일의 변경 사항을 추적하는 '버전 관리 시스템'입니다. (비유:'타임머신', '')

### GitHub
> Git으로 관리하는 프로젝트를 온라인에 백업하고 공유하는 '클라우드 서비스'입니다. (비유: 세이브 파일을 올려두는 '구글 드라이브')

### Git Bash
> 텍스트 명령어를 입력하여 Git을 조작할 수 있게 해주는 '터미널 프로그램'입니다. 마우스 대신 키보드로 명령어를 입력합니다. 터미널 사용법을 익히시고 `IDE`나 폴더 선택 후 `마우스 오른쪽 버튼을 눌러 gitbash로 열기` 등을 통해서 터미널로 보시는 습관을 기르시는게 좋습니다. 

### 커밋 (Commit)
> : 파일들의 현재 상태를 하나의 버전으로 묶어 내 컴퓨터(로컬)에 저장하는 행위입니다. (비유: 현재 상태를 '사진 촬영'하여 저장)

### 푸시 (Push)
> : 내 컴퓨터에 저장된 커밋 내역들을 GitHub(온라인)로 업로드하는 행위입니다. (비유: 사진첩을 클라우드에 '동기화')

### 커밋 메시지 (Commit Message)
> 커밋을 할 때, 어떤 내용을 수정하거나 추가했는지 적어두는 요약 메모입니다.

### README.md
> 프로젝트의 '간판'이자 '사용 설명서' 역할을 하는 마크다운 문서입니다. 프로젝트별로 반드시 작성하는 습관을 기르시는게 좋습니다. AI에게 프로젝트 설명하고 작성해달라고 하면 아주 잘 작성해주니 수정만 해서 쓰시면 됩니다. 프로젝트 최상위에 두면 깃허브 레포지토리의 첫 화면에 표시고 하위 폴더별로도 생성이 가능합니다.
