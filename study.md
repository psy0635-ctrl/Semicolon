# 세미콜론 과제1


## 1. Git이란 무엇이며, GitHub와 어떤 차이가 있는가?

Git은 파일의 변경 이력을 저장하고 관리하는 분산형 버전 관리 시스템이다.  
개발자가 코드를 수정할 때 이전 버전으로 되돌리거나, 변경 내용을 기록하는 데 사용된다.

GitHub는 Git을 기반으로 한 원격 저장소 서비스이다.  
인터넷을 통해 여러 사람이 함께 코드를 공유하고 협업할 수 있도록 도와준다.

정리하면,
- Git: 버전 관리 도구 (로컬에서 사용)
- GitHub: Git 저장소를 올리고 공유하는 플랫폼 (온라인 서비스)

## 2. Markdown(.md) 언어를 사용하는 이유와 사용처

Markdown은 간단한 문법으로 문서를 작성할 수 있는 언어이다.  
복잡한 코드 없이도 제목, 리스트, 강조 등을 쉽게 표현할 수 있다.

사용해야 하는 이유:
- 배우기 쉽고 간단하다
- 가독성이 좋다
- HTML로 변환 가능하다
- 개발 문서 작성에 편리하다

주로 사용하는 곳:
- GitHub README 파일
- 노션(Notion)
- 블로그 (티스토리 등)
- 개발 문서 및 보고서

## 3. 프로그래밍 언어의 종류와 주 사용처

프로그래밍 언어는 목적에 따라 다양한 종류가 있다.

- C언어: 시스템 프로그래밍, 임베디드 개발
- Python: 데이터 분석, 인공지능, 자동화
- Java: 웹 애플리케이션, 안드로이드 앱 개발
- JavaScript: 웹 프론트엔드 및 백엔드
- C#: 게임 개발, 윈도우 프로그램

각 언어는 목적에 따라 사용 분야가 다르며, 상황에 맞게 선택하여 사용한다.

## 4. 라이브러리, 프레임워크, API의 개념과 차이점

라이브러리는 특정 기능을 수행하는 코드의 모음이다.  
필요할 때 가져와서 사용하는 것이 특징이다.

프레임워크는 전체적인 구조를 제공하는 틀이다.  
개발자가 그 틀 안에서 코드를 작성해야 한다.

API는 서로 다른 프로그램이 소통할 수 있도록 만들어진 규칙이다.  
예를 들어, 서버와 클라이언트가 데이터를 주고받을 때 사용된다.

차이점 정리:
- 라이브러리: 내가 필요할 때 가져다 사용
- 프레임워크: 정해진 틀 안에서 개발
- API: 프로그램 간 소통 규칙
